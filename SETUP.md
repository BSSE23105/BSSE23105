# 🚀 Setup Guide — Publishing Your GitHub Profile

Follow these steps in order. Total time: ~10 minutes.

## 1. Create the special profile repository

1. Go to [github.com/new](https://github.com/new) (make sure you're logged in as **BSSE23105**).
2. Repository name: exactly **`BSSE23105`** (same as your username — this is what makes it a *profile* README).
3. Visibility: **Public**.
4. Do **NOT** initialize with a README (we already have one).
5. Click **Create repository**.

## 2. Push this folder

Open a terminal inside the `github-profile` folder and run:

```bash
git init
git add .
git commit -m "feat: initial profile README"
git branch -M main
git remote add origin https://github.com/BSSE23105/BSSE23105.git
git push -u origin main
```

> If git asks you to log in, use your GitHub username and a Personal Access Token
> (create one at Settings → Developer settings → Personal access tokens → Tokens (classic) → `repo` scope).

## 3. Run the 3D contribution graph action (once)

The **Contribution City — 3D** image is generated by GitHub Actions and won't exist until the workflow runs once:

1. Open your repo → **Actions** tab.
2. If prompted, click **"I understand my workflows, go ahead and enable them"**.
3. Select **GitHub-Profile-3D-Contrib** in the left sidebar → **Run workflow** → **Run workflow**.
4. Wait ~1 minute. The action commits `profile-3d-contrib/profile-night-rainbow.svg` and the broken image on your profile becomes the 3D city. It auto-refreshes daily at 08:00 PKT.

*(The push you did in step 2 also triggers it automatically — either way works.)*

## Troubleshooting

| Problem | Fix |
|---|---|
| 3D graph shows broken image | Actions tab → run **GitHub-Profile-3D-Contrib** manually (step 3) |
| Action fails with permission error | Repo → Settings → Actions → General → Workflow permissions → **Read and write permissions** |
| Stats / top-languages cards break | The **official** `github-readme-stats.vercel.app` instance is currently paused, so the README uses the `-sigma-five` community mirror. If that mirror ever dies too, self-host your own free instance in 2 minutes: [one-click Vercel deploy](https://github.com/anuraghazra/github-readme-stats#deploy-on-your-own) → then replace the mirror domain in README.md with your deployment's domain |
| Banner not animating | GitHub caches images via camo — hard-refresh (Ctrl+F5); animations are SVG-native and will work |

> ℹ️ **Why no trophy shelf?** `github-profile-trophy.vercel.app` is currently disabled upstream (`DEPLOYMENT_DISABLED`) — verified during setup. The summary-cards (language donuts + productive-time) replace it with services that actually work.
